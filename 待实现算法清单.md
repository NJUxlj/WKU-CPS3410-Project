### 4.3 Genetic Algorithm (GA)

**目录结构**:
```
genetic_algorithm/
├── __init__.py
├── chromosome.py          # 城市序列编码
├── fitness.py             # 适应度函数
├── selection.py           # 轮盘赌 / 锦标赛 / SUS
├── crossover.py           # OX / PMX / CX
├── mutation.py            # 交换 / 倒位 / 插入
├── genetic_algorithm.py   # 主循环 + 精英策略
└── experiment.py
```

**核心设计**(对齐 GA report 内容):
- **编码**:城市排列(permutation)
- **适应度**:`f = N_cities / tour_cost`(report 公式)
- **选择**:Roulette Wheel + 20 个 Elitism
- **交叉**:片段交叉(报告中示意),扩展为 OX / PMX
- **变异**:random swap(基础) + 5 种邻域变体(1-opt/2-opt/3-opt/City Insertion/2-opt)— 与 report 实验对应
- **种群**:100 个个体,迭代至收敛
- **评估**:与 SA 同标准 — 解质量、收敛时间、稳定性

**复用**:直接 import `simulated_annealing.tsp_instance.load_distance_matrix` 读取 10/50 城市数据

### 4.4 Variable Neighborhood Search (VNS / RVNS)

**目录结构**:
```
vns/
├── __init__.py
├── vns.py                 # Basic VNS
├── rvns.py                # Reduced VNS
├── neighborhoods.py       # 邻域族 N_1, N_2, ..., N_kmax
└── experiment.py
```

**核心设计**(对齐 PPT Slide 5/9 描述):
- **邻域族**:N_1=1-opt, N_2=2-opt, N_3=3-opt, N_4=swap, N_5=insert(可扩展)
- **VNS 主循环**:
  1. 初始解 x
  2. 重复直到停止:
     - 随机选邻域 N_k
     - 在 N_k 中找最优 x'(shaking)
     - 以 x' 为起点做 Local Search 得 x''
     - 若 f(x'') < f(x):x = x'',k=1;否则 k+=1
- **RVNS**:Local Search 步骤替换为随机采样(无爬山,只 shaking)— 与 PPT Slide 5 描述一致
- **评估**:对比 VNS vs RVNS vs GA 在 4 个维度上的差异

**复用**:`local_search.Neighborhood` 接口直接作为 N_k 使用

### 4.5 Tabu Search (TS)

**目录结构**:
```
tabu_search/
├── __init__.py
├── tabu_list.py           # 禁忌表 + 藐视准则
├── neighborhood.py        # 1-opt 邻域(对齐 report)
├── tabu_search.py         # 主循环
└── experiment.py
```

**核心设计**(对齐 Report 内容):
- **编码**:城市序列
- **邻域**:1-opt(报告中明确,O(n²) 复杂度)— 也可扩展 2-opt
- **禁忌表**:`tabuList[i][j] = tabu_duration`,记录被禁忌的交换 (i, j)
- **藐视准则(aspiration)**:若禁忌解优于 best-so-far,允许接受
- **参数**(参考 report):
  - tabu_size = 200
  - region_iteration = 50
  - neighborhood_iteration = 100
  - tabu_duration ∈ [1, 10]
- **评估**:50 城市实例、1000 次重启的解分布、稳定性

### 4.6 Ant Colony Optimization (ACO)

**目录结构**:
```
ant_colony/
├── __init__.py
├── pheromone.py           # 信息素矩阵 τ_ij
├── heuristic.py           # 启发式信息 η_ij = 1/d_ij
├── ant.py                 # 单只蚂蚁构建路径
├── aco.py                 # ACS / MMAS 主循环
└── experiment.py
```

**核心设计**:
- **信息素更新**:蚁周模型(Ant-Cycle)τ_ij ← (1-ρ)τ_ij + Σ Δτ_ij^k
- **路径构建**:状态转移概率 p_ij^k = [τ_ij]^α · [η_ij]^β / Σ [...]
- **参数**:
  - α=1, β=2(信息素/启发式权重)
  - ρ=0.1(挥发率)
  - ants = 30
  - iterations = 500
- **变体**:可对比 AS / ACS / MMAS

### 4.7 Bee Algorithm (BA)

**目录结构**:
```
bee_algorithm/
├── __init__.py
├── employed_bee.py        # 引领蜂(雇佣蜂)
├── onlooker_bee.py        # 跟随蜂
├── scout_bee.py           # 侦查蜂
├── artificial_bee_colony.py  # ABC 主循环
└── experiment.py
```

**核心设计**(Artificial Bee Colony, ABC):
- **食物源**:等价于一个 TSP 路径解
- **三类蜜蜂**:
  - 雇佣蜂(Employed Bees):在邻域内搜索新解,N 个食物源 → N 个雇佣蜂
  - 跟随蜂(Onlooker Bees):根据适应度概率选择食物源继续搜索
  - 侦查蜂(Scout Bees):若某解连续 `limit` 次未更新 → 随机生成新解替换
- **邻域**:与 GA 类似的 swap / 2-opt
- **参数**:
  - colony_size = 50
  - limit = 100
  - max_iterations = 1000
- **评估**:与 SA/GA/TS/VNS 同标准

## 5. 公共基础设施(一次性建设,所有算法复用)

为避免重复造轮子,先建一个共享层:

```
common/
├── __init__.py
├── tsp_instance.py        # 已有(simulated_annealing/)→ 升级到 common/
├── distance.py            # 距离矩阵 + tour cost 增量计算
├── neighborhood.py        # 统一邻域接口(1/2/3-opt, swap, insert)
├── evaluation.py          # 统一评估指标(quality/converge_time/stability)
├── visualization.py       # 收敛曲线 / 路径图 / Box plot
└── data/
    ├── distances_between_cities_10.txt
    └── distances_between_cities_50.txt
```

**统一评估指标**(对应 PPT 4 个维度):
- `quality`:100 次重启中最优 cost
- `convergence_time`:到达 known-optimal 阈值(10→3473, 50→5644)的迭代次数 / CPU 时间
- `stability`:100 次重启 cost 的标准差
- `diversification`:搜索过程中不同解的覆盖率(可用 unique tours / 熵)
- `intensification`:最优解附近邻域内的搜索比例
